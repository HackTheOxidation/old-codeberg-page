Skills/Interests

Most of the skills that I have acquired primarily come from University and (mostly) free-time hobby coding, but in spring of 2022 I got an internship working on a (quite large) python project, so now I also have real-world professional developer experience. Yay!

Operating Systems

  • Linux - I have tinkered with most of the main distros: Debian-/Ubuntu-based, Fedora/RHEL, Arch-based, Gentoo-based etc. and some niche stuff: Artix, GNU Guix, Void Linux, etc. I am currently running Arch as my main distro.
  • BSD - I have also had my hands in some of the BSD-derivatives, mainly FreeBSD and OpenBSD.

Programming Languages

  • Python - My first internship was all about python (django, specifically), which means that I have "Professional" python development experience. The code I wrote for the 'os2datascanner' project during my internship is fully open source! Check it out here: os2datascanner Aside from that, I have used python in non-professional capacity as well, mostly in the form of small scripts and config files, but now I am writing bindings for my moonalloy project: python-moonalloy. In case your are familiar with the UniProt protein database, I wrote a CLI for fetching data: uniprot-cli.
  • C - I have around 1.5 years under my belt from university and hobby coding. I wrote a minimalist web-server for a project: ficus
  • C++ - I know advanced parts of the language from newer standards (C++17/20). I wrote a small JSON-parser from scratch: picoJSON. And a library for parsing .INI-files: libini.
  • Haskell - This language is awesome! Being purely functional, it challenges the way you think and makes you a better programmer (I think). I wrote a few of the challenges from Advent of Code 2021 in haskell: advent-of-code2021.
  • Rust - I have put a reasonable effort into learning it and it has quickly become one of my favorite programming languages. I have written a small 'ls'-clone: neols (crates.io). I am also working on a numpy-like library written in rust with an API in Python and Lua (still in early development): moonalloy
  • Go - I generally find Go to be a nice, minimalistic language. I have done some tinkering with it and written (a very incomplete) cli-based http-client: twig. I also began working on how to extract text from a PDF at work.
  • HTML, CSS - Used for making the previous version of this retro, non-bloated website. Now, everything is written in the awesome org mode.
  • Emacs Lisp - It's just cool and retro and I can make my editor do anything with it. I have contributed a few lines to Silex/docker.el.

And there are some more languages, but I have an opinion about these:

  • JavaScript/TypeScript - I know enough to survive, but I don't really enjoy it…
  • C# - No. Just No…
  • Anything proprietary - Seriously, open source is the way to go.
